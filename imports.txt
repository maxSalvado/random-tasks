Product Backlog Item – User Story
Título:
Gerenciar POCs e Oportunidades com autenticação e controle de permissões via Entra ID

Descrição:

Eu, como editor autorizado do catálogo de serviços, quero acessar o sistema usando minha conta corporativa via SSO (Entra ID) e ter permissão para criar, editar e visualizar registros de POCs e Oportunidades, para que eu possa manter as informações atualizadas e consistentes, garantindo que apenas usuários autorizados possam modificar o conteúdo.

Critérios de Aceitação:

Autenticação

O sistema deve permitir login apenas para usuários autenticados via SSO (Entra ID).

Usuários não autenticados devem ser redirecionados para a tela de login.

Controle de Acesso (RBAC)

Somente membros do grupo de editores configurado no Entra ID podem criar ou editar POCs e Oportunidades.

Usuários sem permissão de edição podem apenas visualizar as informações.

Funcionalidade

Listar todos os registros de POCs e Oportunidades disponíveis no banco de dados.

Buscar um registro específico por ID.

Criar um novo registro, informando todos os campos obrigatórios.

Editar registros existentes, atualizando apenas os campos necessários.

Exibir mensagens de sucesso ou erro para cada ação (criação, edição, busca).

Integração

As operações devem refletir no banco de dados (Cosmos DB, contêiner content) imediatamente após execução.

As APIs devem validar se o usuário está autenticado e autorizado antes de executar qualquer operação.

Validação

Todos os campos obrigatórios devem ser verificados antes de salvar um registro.

IDs duplicados ou inexistentes devem retornar mensagens claras para o usuário.

Definição de Pronto (DoD – Definition of Done):

 Autenticação via Entra ID funcionando no ambiente de testes e produção.

 RBAC configurado no Entra ID e validado no backend.

 Endpoints de CRUD para POCs e Oportunidades implementados e testados.

 Telas de listagem, criação e edição integradas ao backend.

 Feedback visual (sucesso/erro) implementado no frontend.

 Testes manuais e automatizados executados e aprovados.

 Documentação técnica atualizada no repositório.

Tarefas sugeridas:

Configurar grupos e permissões no Entra ID (RBAC).

Implementar autenticação via MSAL no frontend Angular.

Criar endpoints no backend (Azure Functions) para CRUD de POCs e Oportunidades.

Integrar frontend e backend para listagem e criação/edição de registros.

Implementar validações de formulário no frontend.

Criar testes unitários e de integração.

Atualizar documentação e instruções no repositório GitHub.
